# lawiki v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 交付一个可被不同 agent 识别使用的 skill（lawiki），把法律案件资料整合成可控、可溯源的案件 wiki。

**Architecture:** 纯 skill 包，无代码、无 CLI。`skill/lawiki/SKILL.md` 是短而硬的编排+铁律指令；`skill/lawiki/templates/case-wiki/` 是 agent 拷贝即用的案件脚手架，其中 `CLAUDE.md` 承载详细页面格式（第二层约束）。上游 makeitdown（独立 CLI）负责文件→md，由 agent 照 skill 调用。

**Tech Stack:** Markdown（skill / 模板 / 文档）、git。验证靠结构检查（PowerShell/grep）+ 样例案件端到端干跑 + 验收清单。无 Python、无 pytest。

> 设计依据：`docs/superpowers/specs/2026-06-16-lawiki-design.md`（已批准）。

## Task 2: 案件 schema —— templates/case-wiki/CLAUDE.md

承载第二层约束：四类页面的权威格式 + 铁律 + 锚点 + 事实/分析隔离 + 矛盾处理。在案件目录内工作时自动加载。

**Files:**
- Create: `skill/lawiki/templates/case-wiki/CLAUDE.md`

- [ ] **Step 1: 写 CLAUDE.md**

文件 `skill/lawiki/templates/case-wiki/CLAUDE.md` 内容：

````markdown
# 本案件 wiki 维护规范（schema）

你在维护一个法律案件的 wiki。三层结构（前两层不可变，你只写第三层）：
- `原始资料/`：原件，真相之源，永不修改。
- `_md/`：makeitdown 转换产物，来源层，永不修改。
- `wiki/`：本目录，你拥有并维护。

## 五条铁律（与 lawiki skill 一致，不可违反）

1. **零来源不可写**：每句事实必挂锚点；无来源不许写进 wiki。
2. **不推断不脑补**：只写文档明确说的；分析/推论必须显式标为「分析」，与事实物理隔离。
3. **可疑来源标未核验**：`_md` 的 frontmatter 含 `quality: suspect` 的来源，引用一律标「未核验」，绝不当既定事实。
4. **矛盾只暴露不调和**：新旧冲突时标记冲突，绝不静默覆盖。
5. **要害逐字照录**：日期、金额、当事人名、条款原文逐字引用，不转述。

## 引用锚点

固定格式（不使用页码）：

```
〔来源: _md/<相对路径>：「<逐字上下文片段>」〕
```

- 片段取自源 md 原文，逐字，足以让人凭上下文回原件定位。
- 案件主体的每条属性、每个法律事实、每条时间线，都必须挂锚点。
- 可疑来源在锚点后追加「（未核验）」。

## 事实与分析的隔离（铁律 2）

- 事实：直接陈述 + 锚点。
- 分析：只能放在页面的 `## 分析` 小节内，或 `> 分析：…` 引用块里，显式标注，绝不与事实混排。

## 板块与页面格式

### 案件主体/<主体名>.md
每个主体一页。收录在 `_md` 中能找到的、与该主体有关的一切：身份信息（如身份证号）、代理人/法定代表人、主体资格、关联关系等。

```markdown
---
类型: 案件主体
更新于: <YYYY-MM-DD>
---
# <主体名>

- 身份信息：<…> 〔来源: _md/…：「…」〕
- 代理人/法定代表人：<…> 〔来源: _md/…：「…」〕
- 关联关系：<…> 〔来源: _md/…：「…」〕
```

### 法律关系/<关系名>.md
按**中国大陆法律**确定案件归属类别，每组法律关系一页：商事（股权/合同/商标/知产纠纷…）、民事（借贷/房产纠纷…）、刑事（按罪名）。

```markdown
---
类型: 法律关系
更新于: <YYYY-MM-DD>
---
# <关系名，如：民间借贷纠纷>

- 类别：<商事/民事/刑事 — 具体案由>
- 主体：[<甲>](../案件主体/<甲>.md)、[<乙>](../案件主体/<乙>.md)
- 请求权基础/法律依据：<…>
- 相关法律事实：[<事实>](../法律事实/<事实>.md)
```

### 法律事实/<事实名>.md
有法律意义的事实点，每个一页，**必须挂证据支撑**（如"欠债事实"←欠条+转账记录）。

```markdown
---
类型: 法律事实
更新于: <YYYY-MM-DD>
---
# <事实名，如：甲向乙借款 50 万元>

- 事实：<逐字要害，含金额/日期等> 〔来源: _md/…：「…」〕
- 证据：<欠条/转账记录…> 〔来源: _md/…：「…」〕

## 分析
> 分析：<法律意义>（如有；无则省略本节）
```

### 时间线/总览.md
把各法律事实按时序串起来，每条指回事实页与来源。

```markdown
---
类型: 时间线
更新于: <YYYY-MM-DD>
---
# 时间线

- <YYYY-MM-DD> <事件> → [<事实>](../法律事实/<事实>.md) 〔来源: _md/…：「…」〕
```

## index.md / log.md 维护

- 每次 ingest 后更新 `index.md`：按板块列页面 + 一行摘要。
- 向 `log.md` 追加：`## [YYYY-MM-DD] ingest | <来源文件名>`。

## 矛盾处理（铁律 4）

新来源与已有页面冲突时，在相关页面加 `## ⚠ 冲突` 小节，并列冲突双方与各自锚点，**不覆盖**原内容。
````

````markdown
name: lawiki
description: Use when building or maintaining a Chinese legal-case knowledge wiki from a folder of case materials — drives makeitdown to convert raw documents to markdown, then files them into a controlled, source-anchored case wiki (案件主体/法律关系/法律事实/时间线). Triggers on 整理案件资料、建案件库、ingest case files, or build a case wiki.
---

# lawiki — 法律案件 wiki 构建

把一个案件的原始资料整合成可控、可溯源的 wiki。你（agent）负责全部归档与维护。

## 何时用
用户把法律文件放进某案件的 `原始资料/`，要求"处理 / 整理 / 建库"时。

## 流水线
```
原始资料/ ──makeitdown──▶ _md/ ──ingest──▶ wiki/
```

## 第一步：确保案件结构存在
若案件目录下没有 `wiki/`，照模板脚手架创建：
1. 把本 skill 的 `templates/case-wiki/` 整个拷到案件目录、改名为 `wiki/`。
2. 确保案件目录下有 `原始资料/` 与 `_md/`（无则建空目录）。

不要即兴改动结构——固定结构保证可复现。

## 第二步：转换（调 makeitdown CLI）
makeitdown 是独立上游工具。在案件目录执行：
```
makeitdown 原始资料 -o _md
```
转换后读 `_md/report.json`，留意 `warned`/`failed`/`skipped`。失败或跳过的文件**不要凭空补内容**，按缺失处理并告知用户。

## 第三步：ingest（逐个来源归档进 wiki）
对 `_md/` 下每个 `.md`：
1. 读其正文与 frontmatter。
2. 若 frontmatter 含 `quality: suspect` → 该来源的所有引用标「未核验」。
3. 抽取与案件主体相关的信息 → 写/更新 `wiki/案件主体/<主体名>.md`。
4. 提炼有法律意义的事实点及其证据 → 写/更新 `wiki/法律事实/<事实名>.md`。
5. 判定/更新案件的法律关系 → `wiki/法律关系/<关系名>.md`。
6. 把事实按时序并入 `wiki/时间线/总览.md`。
7. 维护交叉引用，更新 `wiki/index.md`，向 `wiki/log.md` 追加一条。

各页面的**具体格式**见案件 `wiki/CLAUDE.md`（在案件目录内工作时自动加载）。

## 五条铁律（不可违反）
1. **零来源不可写**：每句事实必挂锚点 `〔来源: _md/<路径>：「逐字上下文片段」〕`；无来源不许写进 wiki。
2. **不推断不脑补**：只写文档明确说的；分析/推论必须显式标「分析」，与事实物理隔离。
3. **可疑来源标未核验**：`quality: suspect` 的来源，引用一律标「未核验」，绝不当既定事实。
4. **矛盾只暴露不调和**：新旧冲突时标记冲突，绝不静默覆盖。
5. **要害逐字照录**：日期、金额、当事人名、条款原文逐字引用，不转述。

> 跨 agent：Claude Code / Copilot 用本 `SKILL.md`；Codex 等把案件 `wiki/CLAUDE.md` 视作 `AGENTS.md` 等价物，并把本文件内容作为系统指令喂入。详见 README。
````

```markdown
# lawiki

把一个法律案件的资料整合成**可控、可溯源**的案件 wiki。成果是一个**可被不同 agent 使用的 skill**，不是 CLI 工具。

## 流水线

```
原始资料/ ──makeitdown──▶ _md/ ──lawiki skill──▶ wiki/
```

- `原始资料/`：你丢入的法律文件（不可变）。
- `_md/`：makeitdown 转换出的 markdown（不可变来源层，带 frontmatter 与 report.json）。
- `wiki/`：LLM 按 skill 构建维护的案件 wiki：案件主体 / 法律关系 / 法律事实 / 时间线 + index/log。

## 上游依赖

makeitdown（独立工具）：把各式文件转成 LLM 可读的 md。lawiki 通过其稳定输出契约（frontmatter 的 `source` / `quality: suspect` 等）对接。

## 在不同 agent 下挂载

- **Claude Code / Copilot**：把 `skill/lawiki/` 放入对应的 skills 目录，agent 会按 `description` 自动触发。
- **Codex 等**：把案件 `wiki/CLAUDE.md` 用作 `AGENTS.md` 等价物；并把 `skill/lawiki/SKILL.md` 内容作为系统指令喂给 agent。

## 用法

把文件放进某案件的 `原始资料/`，让 agent "处理/整理/建库"。agent 会调 makeitdown 转换、再按 skill 把来源归档进 `wiki/`。无需手动跑中间步骤。

## 可控性（为何可信）

skill 内嵌五条不可违反的铁律：零来源不可写、不推断不脑补、可疑来源标未核验、矛盾只暴露不调和、要害逐字照录。每条事实挂固定格式来源锚点 `〔来源: _md/…：「逐字片段」〕`，可回溯、可机检。

## 设计与计划

- 设计：`docs/superpowers/specs/2026-06-16-lawiki-design.md`
- 实现计划：`docs/superpowers/plans/2026-06-16-lawiki-v1.md`
```
