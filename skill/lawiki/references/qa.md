# 案件问答（交叉验证：wiki 结论 × RAG 原文）

> 用户就**已建好的案件**提问时走本协议。核心：同时取 wiki 已综合结论与 RAG 原文证据——**一致则答；不一致则查因；查不出则把两套答案 + 各自原始依据并列交用户裁决**。继承三类标注纪律（EXTRACTED / INFERRED / AMBIGUOUS）。
>
> RAG 是可降级旁路：检索经 `<SKILL_DIR>/tools/rag.py`（见 `rag.md`）。没装 / 没建索引 / 模型不一致 → 退化为「仅 wiki」，并明确告知用户「当前无 RAG 交叉验证」。

## 触发

「问本案…」「关于这个案子…」「本案里 X 是什么/是多少/是谁」等**就案件内容提问**——与「整理资料 / 建库 / ingest」这类**构建**触发词区分。构建走 SKILL.md 流水线，提问走本协议。

## 第一步 · 双路并行取证

**两路都要走，不可只凭一边。**

1. **wiki 路**：读 `wiki/index.md` → 顺 `[[wikilink]]`/grep 找相关页 → 取其**已综合结论 + 既有锚点**。
2. **RAG 路**：
   ```
   python <SKILL_DIR>/tools/rag.py search <案件根> "<问题>" -k 8
   ```
   得到原始 `_md/` 片段 + 相对路径 + 逐字 `text` + `score` + `quality` + 现成 `anchor`。
   - `k≥8` 防假冲突：先凑够上下文再判，别把「RAG 没检索到」误判成「矛盾」。
   - 若返回 `rag_available:false` → 跳到「仅 wiki」降级（见末节）。

## 第二步 · 比对，四情形分流

把 wiki 结论与 RAG 逐字原文对齐，落入四者之一：

- **① 一致** → 直接回答；**同时**引用 wiki 页（`[[页名]]`）与 `_md/` 锚点，标注「wiki 与原文互证」。
- **② wiki 沉默 / 未覆盖**（≠冲突）→ 用 RAG 原文回答，挂 `_md/` 锚点，标 `> [!note] 分析`「该点尚未入 wiki」；可建议用户是否 ingest 进 wiki。
- **③ 不一致，能定因** → 查因三连：
  1. **wiki 总结失真？** 比对 wiki 断言 vs RAG 逐字原文（拔高/转述/张冠李戴）。
  2. **RAG 漏上下文？** 提高 `-k` 或换检索词再查。
  3. **源文件本身冲突？** 看 wiki 是否已有 `> [!warning] ⚠ 冲突` callout。
  
  定因后**以 EXTRACTED 逐字原文为准**作答，并指出 **wiki 何处需修**（交用户确认后才回填，见 SKILL.md）。
- **④ 不一致，查不出因**（安全阀）→ **不许静默取舍**：把【wiki 答案 + 锚点】与【RAG 答案 + `_md/` 锚点】**并列**，各附相对路径 + 逐字片段，明说「**无法判定，请人工溯源裁决**」→ 交用户。

## 第三步 · 引用纪律（继承铁律）

- 每个事实陈述挂 `〔来源: _md/…：「逐字原文」〕`；逐字片段取自 RAG 返回的 `text`（或直接用 `anchor`，通常缩到具体支撑句）。**一条断言 ← 一条能完整支持它的引文。**
- 来源 `quality: suspect` 的（wrapper 已在 `anchor` 后缀「（未核验）」/置 `unverified:true`）→ 引用时保留该标注，绝不当既定事实。
- 推断标 `> [!note] 分析`；冲突/存疑标 AMBIGUOUS（`> [!warning] ⚠ 冲突` 并列各方 + 锚点）。
- **「一致/不一致」是 LLM 判断、会错** → 永远把两边逐字原文摆出供用户复核，不要只给「一致」二字。
- 问答引用机器可校验：用户认可的问答结论已是 wiki 锚点格式，可直接回填 wiki，并能过 `lint check`。

## 共同盲区与降级

- wiki 与 RAG 都从 `_md/` 派生，**抓不出 `_md/` 自身的 OCR 错** → 靠 `quality: suspect` 标注 + 必要时回溯 `原始资料/` 原件兜底。
- **仅 wiki 降级**：RAG 不可用时，照常用 wiki 作答并挂既有锚点，但须对用户挑明「**当前无 RAG 交叉验证，结论仅来自已综合的 wiki**」，建议其装好 RAG / 重建索引后复核要害点。
